# Security Policy

GreyNOC Detections is for defensive detection engineering only. Rules and examples should help defenders identify suspicious activity in authorized environments.

Do not submit exploit code, offensive playbooks, credential material, private customer data, or instructions intended to enable unauthorized access.

## Reporting Security Issues

Please report suspected security issues privately to the maintainers. Include:

- A short description of the issue.
- Affected files or rule IDs.
- Safe reproduction steps, if applicable.
- Any public references that help with defensive context.

Do not open public issues containing secrets, exploit details, or customer-specific telemetry.
