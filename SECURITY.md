# Security Policy

GreyNOC Detections is for defensive detection engineering only. Rules and examples should help defenders identify suspicious activity in authorized environments.

## Supported Scope

Security review covers the current default branch of this repository unless GreyNOC documents additional supported release branches.

## Reporting Security Issues

Please do not open public issues containing secrets, exploit details, customer-specific telemetry, credentials, private data, or working attack instructions.

Use GitHub private vulnerability reporting when it is available for this repository. If a private report path is not available, open a minimal public issue that says you have a security report for @GreyNOC and omit sensitive details until a private channel is arranged.

Include:

- A short description of the issue.
- Affected files or rule IDs.
- Safe reproduction steps, if applicable.
- Any public references that help with defensive context.

Do not submit exploit code, offensive playbooks, credential material, private customer data, or instructions intended to enable unauthorized access. Do not access, modify, destroy, or exfiltrate data that is not yours. Good-faith security research and defensive reports are welcome.
